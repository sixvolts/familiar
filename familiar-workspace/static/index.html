        Shards let you create purpose-built assistants inside Familiar, each with their own prompt, tools, and memory.

        Name ID Mode Visibility Tools Tokens Status Updated

        Identity

        What this shard is called and what it does.

        Lowercase letters, numbers, hyphens. Immutable after create.

        Behavior

        How the shard persists and where its memory shows up.

        Ephemeral: no memory writes, no sessions. Persistent: writes memory and keeps sessions.
        Isolated: memory hidden from top-level Familiar. Promoted: surfaced to both views.
        Advanced model settings
        Defaults to shard:<id>. Every memory the shard writes is tagged with this.
        An explicit model ID bypasses the router. Blank = router selection.
        Mutually exclusive with model preference.

        Sign-in & access

        What a shard-passkey login can reach. Enroll devices after saving.

        When off, no passkey on this shard can mint a session — keeps it purely API-driven.
        Leave all unchecked to inherit every panel you can see. Checked = explicit allow-list.
        Tick books the shard can read/write. Leave all unchecked to inherit your memberships.
        Sessions slide — a device in use stays signed in; login expires only after this much idle time. Blank = gateway default (2h). A kiosk might use 604800 (a week).

        Tools and skills this shard can use.

        Tools this shard can invoke. Write-capable memory tools are disabled for ephemeral shards.

        Scheduled Actions

        Have your Familiar perform actions for you in the background on a scheduled basis. You define the work to do, the data that's in scope, and how Familiar responds.

        Name Trigger Envelope Reports to State Last run Last fired
        Loading
        New Action

        Definition

        What the action is called and what it does.

        Runs through the full pipeline — the model can use its tools (read pages, search, etc.) before reporting.

        Trigger

        When the action fires.

        Presets: daily 7am · hourly · Mondays 9am
        Every save in this book fires the action — autosave bursts are throttled by the interval below.
        A secret webhook URL is generated when you save; POST to it to fire this action.

        Report

        Where the result is delivered.

        Reports DM you on Slack via your linked Slack identity. Requires a Slack identity link (Users panel) and the bot's im:write scope.
        A dedicated "Scheduled: …" conversation is created with the action; reports land there as assistant messages you can reply to.
        Pings your installed Familiar app when the action runs (in addition to the destination above); tapping opens its run history. Add Familiar to your home screen and enable notifications in Account.

        Envelope & limits

        How the run is scoped and capped.

        "Run as you" uses your memory, prompt, and full toolbox. Ephemeral runs the prompt bare: no memory, no tools, nothing written. A shard bounds the run with its tool allowlist, memory scope, and pinned model.
